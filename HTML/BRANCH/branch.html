O que são Branch's

Branch é uma maneira de "separar" as versões do nosso projeto.

Quando iniciamos um repositório no Git, ele sempre será iniciando na

branch/main

É recomendável não utilizar a

branch/main
como repositório de modificações constantes. Sempre crie outras para testar outros códigos e features. Deixando ela sempre como um
backup de segurança
.

Depois de pronto e testado todas as novas funcionalidades, temos a possibilidade de

mesclar
as branchs.

Criando uma nova Branch

Podemos criar novas branchs quisermos, basta usar o comandos.

O git deixa um * ao lado da branch que está ativa no momento.

* main

  • git branch nome_da_branch
    Esse comando, criamos umas nova.
  • git branch
    Usamos para ver todas as branchs ativas.
  • git checkout nova_branch
    Usamos para trocar entre as branchs.
  • git checkout -b nova_branch
    Com esse comando, criamos uma nova branch e já mudamos pra ela.

Podemos deletar uma branch, mas é altamente recomendável que mantenhamos um histórico de versões dos nossos projetos.

  • git branch -d nome_da_branch
    Serve para deletar uma branch criada.
Unindo Branchs

Depois de ter feito as alterações na branch paralela, agora podemos unir as branchs. Para isso precisamos estar na branch que vai receber as alterações.

  • git merge nome_da_branch
    usamos para unir as branchs.
Voltar ao topo
Git Stash

Podemos resetar uma branch para um estado inicial usando o Stash. Stash é como um ponto de recuperação. Podemos criar vários para poder voltar neles quando necessário.

Devemos fazer as alterações primeiro, para poder gerar um Stash. Assim teremos um ponto onde voltar.

Vamos usar o comando

git stash
na branch que queremos resetar.

Podemos reverter um Stash acessando a lista de Stash's. Podemos acessar pelo

ID {0}
que vai estar entre { }.

  • git stash list
    vai listar todos os stashs criados.
  • git stash list
    vai listar da primeira pra última, lembrando que o indice 0 é a mais recente.
  • git stash apply 0
    é o comando para acessar o stash que queremos voltar, lembrando que o ponto de recuperação é o numero do Stash.

Podemos verificar também, as diferenças entre as Stash.

  • git stash show -p 0
    vai mostrar o que há na Stash.
